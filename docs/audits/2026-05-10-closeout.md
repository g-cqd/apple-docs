# Audit closeout — 2026-05-10

**Scope:** remediation pass on the long tail of findings from the five 2026-05-09 audits (`2026-05-09-overview.md`, `…-strict-architectural.md`, `…-strict-security-sota.md`, `…-findings-brief.md`, `…-deep-exhaustive.md`).

The big rocks (god-module decomposition, snapshot tar jail, MCP origin policy, body caps, FK enforcement, SSRF gate, render-cache TTL, reader-pool bounds, lifecycle handlers, Prometheus, CodeQL, dependabot, SF Symbol pre-render + validation) were already fixed before this pass. This closeout describes what was fixed in this pass and what intentionally remains.

## Fixed in this pass

### Phase A — security & defaults

- **A1** `src/web/serve.js:104` — `Bun.serve` now binds `127.0.0.1` by default (was `*`/all-interfaces). Honors `--host` and `APPLE_DOCS_WEB_HOST` for explicit LAN exposure.
- **A2** `cli.js`, `src/cli/help.js` — plumbed `--host` and `--rate-limit` through `web serve`.
- **A3** `src/web/serve.js:46-58` — per-client-IP token-bucket gate is now **disabled by default**. Opt back in via any of: `APPLE_DOCS_WEB_RATE`, `APPLE_DOCS_WEB_BURST`, `APPLE_DOCS_WEB_RATE_LIMIT=1`, or `--rate-limit`. The narrow `/docs/<key>` per-IP SSRF gate inside `docs.route.js` (A7) stays on — it's a specific amplifier control, not a general rate limit.
- **A4** New `src/web/csp.js` — emits hash-based Content-Security-Policy. The single inline IIFE on the 404 page (`src/web/templates/not-found.js`) is hashed at module load (`Bun.CryptoHasher`); `default-src 'self'` for everything else; `frame-ancestors 'none'`; `object-src 'none'`. Two `<script type="application/json">` data blocks (fonts, framework tree) are unaffected — browsers don't execute them.
- **A4** `src/web/context.js:164-171` — added CSP and tightened `Cross-Origin-Resource-Policy` from `same-site` to `same-origin`.
- **A5** `src/mcp/http-server.js:12-22` — added `Permissions-Policy`, `Cross-Origin-Opener-Policy: same-origin`, `Cross-Origin-Resource-Policy: same-origin` to MCP HTTP `SECURITY_HEADERS` for parity with the web server. CSP is intentionally web-only (MCP is JSON-RPC, no HTML).

### Phase B — reliability hardening

- **B1** `src/content/render-html/markdown.js` — `markdownToHtml` blockquote recursion is now bounded at `MARKDOWN_MAX_BLOCKQUOTE_DEPTH = 32`. Past the cap, the inner text falls through to a single-paragraph `<blockquote>` instead of recursing further. Verified with a 50-deep adversarial input — 33 blockquote opens (32 levels + 1 flat fallback), no stack overflow.
- **B2** `src/lib/per-host-rate-limiter.js` — `buckets` Map is now an insertion-ordered LRU capped at 256 (override via `APPLE_DOCS_HOST_BUCKET_MAX`). Cache hit re-inserts to move-to-front; once the cap is exceeded the oldest hostname is evicted. Two new unit tests cover the eviction path.
- **B3** `ops/bin/watchdog.sh:55` — the daemon liveness probe now hits `/readyz` (DB + reader-pool check) instead of `/healthz` (accept-only). A wedged DB is now caught by the watchdog.

### Phase C — code quality / cognitive load

- **C1** New `src/lib/run-step.js` — labeled async-phase helper with structured logging + duration accounting. Distinct from `safe-call.js` (generic error boundary). Six tests.
- **C2** `src/commands/sync.js` — refactored four ad-hoc `try { … } catch (e) { logger.warn(…) }` phases through `runStep`. Behavior preserved (each phase still continues on error and contributes to `failedSources`); orchestrator is roughly 30 lines lighter and the failure-handling pattern is now uniform.
- **C3** `src/web/build.js` — links-audit phase migrated to `runStep`. Other phases are intentionally fail-fast (one outer try/catch) and don't need wrapping.
- **C4** Investigated `consolidate.js` — already fail-fast with no ad-hoc `try { … } catch (e) { logger.warn(…) }` phase blocks. No-op.
- **C5** `index.js` — collapsed from 51-line standalone bootstrap to a 14-line shim that delegates to `cli.js mcp start`. `cli.js:43` now honors `APPLE_DOCS_LOG_LEVEL` (closes the env-drift hazard between the two binaries). The `apple-docs-mcp` entry point still works identically; passing `--help` even routes through the shared help system.

## Intentionally still out of scope

Two large refactors that the audits flagged (and that I considered for this pass) require dedicated phases:

1. **Kill the `pages` table** — the half-finished `pages → documents` migration noted in `2026-05-09-deep-exhaustive.md §1.5`. Both `src/storage/repos/pages.js` (165 LOC) and `repos/documents.js` are still alive; `getPage()` still returns polymorphic shapes. This is the single largest cognitive-load tax remaining in the storage layer. Scope: schema migration v18, drop `repos/pages.js`, rename `documents.key → path`. Several days of work; defer to its own PR.

2. **Move `_trigramCache` into SQLite** — `2026-05-09-deep-exhaustive.md` perf-win #4. The cache is currently per-process-per-worker and bounded only by a `corpusStamp` invalidator. Moving the trigram → doc-id mapping into a SQLite table eliminates the multi-hundred-MB worst-case footprint at high worker counts. Scope: migration, repo method, hot-path swap; several days.
